# 补天平台漏洞提交规则

## 漏洞类型
- SQL注入
- XSS（反射型/存储型/DOM型）
- 信息泄露
- 未授权访问
- 命令执行
- SSRF
- 逻辑漏洞
- 配置错误

## 审核标准

### 通过条件
1. 漏洞可实际利用
2. 有明确的复现步骤
3. 提供截图/视频证据
4. 影响范围明确
5. 首次发现（非重复）

### 拒绝原因
1. 纯理论漏洞，无实际利用场景
2. 已被公开或已修复的漏洞
3. 自动化扫描工具批量提交
4. Self-XSS（自己才能触发）
5. 仅影响极旧浏览器
6. 社工钓鱼类问题
7. 功能设计如此（By Design）
8. 无法证明实际危害

## 漏洞等级

| 等级 | 说明 | 奖金 |
|------|------|------|
| 严重 | RCE、SQL注入（可获取数据） | 1000-10000 |
| 高危 | XSS（存储型）、SSRF、未授权访问 | 500-5000 |
| 中危 | XSS（反射型）、信息泄露、配置错误 | 200-1000 |
| 低危 | 安全头缺失、版本信息泄露 | 50-300 |

## 提交格式

### 标题格式
```
【漏洞类型】厂商名称存在XXX漏洞
```

### 描述内容
1. 漏洞描述
2. 漏洞URL
3. 复现步骤
4. 证据截图
5. 修复建议
